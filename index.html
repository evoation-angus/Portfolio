Dragonside

Handing and processing Riot Games API data with Java to create Objects ready for database storage. Storing the data in a database in MySQL or PostgreSQL using JDBC. Website with simple functionality to use to find the desired data with GET and POST requests.

Information

This webiste provides information for the video game Leauge of Legends(LoL). All that needs to be know is when playing an attacking role, you will need a supporting role teammate. LoL provides a choice of over 150 characters. This tools helps match damage and support characters based on statistics. It determines a winrate basted on the best players match history.

Riot API and data processing

The first piece of information we need is match data. Since the game format is 5 vs 5, each team will have a winner and a losing damage and support pairing.

To get to the match data I get a list of players from the top rank. Then I get the players PUUID which links their profile to their game history and get MatchID of their 100 latest games. And then finally we can look at the data for 1 game.

Now with the match data I find each players role and character. I look for each teams Support/Utility and Carry/Bottom and pair the winning and loosing pairs into an Object. The object stores the Attacking characters name and a list of all its supporting character Objects. The Support object contains the Support name, win or loss, and frequency of apperance which is needed to calculate the Win rate.
